# 导入服务名称策略更新

## 更新内容

根据用户反馈，修改了导入服务配置功能中的服务名称处理策略。

### 变更前
- 导入服务时自动在原服务名称后添加"-imported"后缀
- 例如：原服务名为"web-api"，导入后变为"web-api-imported"

### 变更后
- 导入服务时保持原始服务名称不变
- 例如：原服务名为"web-api"，导入后仍为"web-api"

## 影响

### 优点
- 保持服务名称的一致性和可读性
- 避免不必要的名称修改
- 更符合用户的直觉预期

### 注意事项
- 如果目标项目中已存在同名服务，导入时会失败
- 用户需要在导入前检查是否存在名称冲突
- 如果遇到冲突，需要先删除或重命名现有服务，或者选择不同的服务进行导入

## 错误处理

当导入过程中遇到服务名称冲突时：
1. 系统会显示具体的错误信息
2. 成功导入的服务数量和失败的服务数量会分别统计
3. 用户可以根据错误信息调整后重新尝试导入

## 建议的使用流程

1. 在导入前，先检查目标项目中是否已存在同名服务
2. 如果存在冲突，可以选择：
   - 删除或重命名现有服务
   - 选择不同的服务进行导入
   - 分批导入，避免冲突
3. 导入完成后，检查导入结果并根据需要调整服务配置
