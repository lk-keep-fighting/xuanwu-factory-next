# Pod 监控功能实现

## 功能概述

实现了 Pod 状态的实时监控功能，用户可以通过点击监听按钮来跟踪部署状态。

## 主要改进

### 1. StatusCard 组件优化
- 移除了默认显示的 Pod 详细状态信息
- 添加了"监控"按钮，只在有 Pod 时显示
- 简化了状态卡片的显示，只显示基本信息和 Pod 数量

### 2. 新增 PodMonitorDialog 组件
- 实时监控 Pod 状态变化
- 每 3 秒自动刷新一次
- 支持手动刷新
- 支持开始/停止监控
- 详细显示每个 Pod 和容器的状态信息

## 功能特性

### 实时监控
- 自动每 3 秒刷新 Pod 状态
- 实时状态指示器（绿色脉冲表示监控中）
- 可手动开始/停止监控

### 详细信息展示
- Pod 名称、状态、阶段
- 容器就绪状态、重启次数
- 容器运行时间、等待/终止原因
- 清晰的视觉层次和状态指示

### 用户体验
- 弹窗形式，不影响主界面
- 响应式布局，支持大屏显示
- 清晰的状态颜色编码
- 便于部署时跟踪状态变化

## 使用场景

1. **部署跟踪**: 用户点击部署后，可以打开 Pod 监控来实时查看部署进度
2. **故障排查**: 当服务出现问题时，可以查看 Pod 和容器的详细状态
3. **运维监控**: 日常运维时监控 Pod 的健康状态

## 技术实现

- 使用 React hooks 管理状态和副作用
- 使用 setInterval 实现定时刷新
- 使用 useRef 管理定时器引用
- 使用 Dialog 组件实现弹窗
- 使用 Badge 和图标进行状态可视化

## 布局改进

```

```

```

```
